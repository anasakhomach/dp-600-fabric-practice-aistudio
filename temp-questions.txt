This question is part of the **Litware, Inc. Case Study (Topic 1)**. To meet the security requirements while following the **principle of least privilege**, you must map the specific needs of each role to the additional permissions available when sharing a Fabric data store (Lakehouse or Warehouse).

### **Correct Answer Selections**

*   **DataEngineers:** Read All SQL analytics endpoint data
*   **DataAnalysts:** Build Reports on the default dataset
*   **DataScientists:** Read All Apache Spark

---

### **Explanation**

1.  **DataEngineers (Read All SQL analytics endpoint data):**
    *   **Requirement:** The case study states that "The data engineers must be able to read from and write to the data store". 
    *   **Logic:** In Microsoft Fabric, sharing a data store with the **Read All SQL analytics endpoint data** permission allows the user to query all tables using T-SQL. This is essential for data engineers to validate data loads and perform the "read" portion of their duties within the data store.
    *   **Constraint:** They must have "No access... to datasets or reports," so they should **not** receive the "Build Reports" permission.

2.  **DataAnalysts (Build Reports on the default dataset):**
    *   **Requirement:** The analysts must have access to "create Power BI reports by using the semantic models".
    *   **Logic:** When you share a data store, selecting **Build Reports on the default dataset** grants the user the "Build" permission on the automatically generated semantic model. This allows them to create new reports in the Power BI service or Desktop using that data source.
    *   **Constraint:** They only need access to the "dimensional model objects," which is handled via the semantic model or RLS.

3.  **DataScientists (Read All Apache Spark):**
    *   **Requirement:** Data scientists "must be able to read from the data store... by using a Spark notebook".
    *   **Logic:** The **Read All Apache Spark** permission specifically enables users to access the underlying files and tables of a data store using the Spark engine. This is the standard permission required for Notebook-based data science workflows in Fabric.

### **Learning Material to Memorize**

To master sharing permissions in Fabric, memorize the **"Access vs. Tool"** mapping:

| Permission Name | Tool / Language Enabled | Role Example |
| :--- | :--- | :--- |
| **Read All SQL analytics endpoint data** | T-SQL / SQL Query Editor | Data Engineer / SQL Developer |
| **Read All Apache Spark** | Python, Scala, Spark SQL / Notebooks | Data Scientist |
| **Build Reports on the default dataset** | Power BI Report Builder / Desktop | Data Analyst |

**Summary: The Principle of Least Privilege**
Always grant the minimum engine access required for the user's tool. If they use Notebooks, give them Spark. If they use SQL Management Studio or the SQL editor, give them SQL. If they only build reports, give them Build permissions on the dataset.

### **Detailed Cross-Reference & Logic**

#### **1. DataScientists (Requirement: Spark Notebooks)**
*   **Source Requirement:** "The data scientists must be able to read from the data store... They will access the data by using a **Spark notebook**".
*   **Technical Logic:** In Microsoft Fabric, notebooks interact with the data store using the Spark engine. When sharing a lakehouse or warehouse, the **"Read All Apache Spark"** permission is the specific entitlement that allows users to read data via Spark sessions and notebooks.
*   **Why not others?** Granting them "Build Reports" would violate the principle of least privilege, as their role is focused on data science modeling, not Power BI report development.

#### **2. DataAnalysts (Requirement: Create Reports)**
*   **Source Requirement:** "The data analysts... must have access to **create Power BI reports** by using the semantic models".
*   **Technical Logic:** When you share a Fabric data store, selecting **"Build Reports on the default dataset"** grants the user the "Build" permission on the underlying semantic model. This is a prerequisite for users to create new reports in the Power BI service or Desktop using that data source.
*   **Why not others?** Analysts at Litware are restricted to "read access to **only the dimensional model objects**". Giving them "Read All Apache Spark" or "Read All SQL" would give them broader access to raw or non-dimensional data, violating their specific access constraint.

#### **3. DataEngineers (Requirement: Read/Write, No Reports)**
*   **Source Requirement:** "The data engineers must be able to **read from and write to** the data store. **No access must be granted to datasets or reports**".
*   **Technical Logic:** While "writing" to a data store typically requires a workspace role (like Contributor), the question asks for the additional permission assigned **when sharing**. To satisfy the "read" requirement while strictly adhering to the "No access to reports" constraint, **"Read All SQL analytics endpoint data"** is the only valid choice. This allows them to use T-SQL to validate the data they have ingested without granting any permissions related to Power BI reports or semantic models.
*   **Why not others?** Granting "Build Reports" is explicitly forbidden by the requirement "No access must be granted to datasets or reports".
